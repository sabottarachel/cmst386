Preprocessing

Explain the ways to implement preprocessing (client-side vs server-side)

To briefly describe the difference, client-side scripting is executed by the web browser and server-side is executed on the web server. One way to implement preprocessing is to use LESS files because they can be compiled in a standard CSS file. Since it is browser friendly, it is client-side scripting since it is executed by the browser. The preprocessing action is done by JavaScript, which is still a client-sider server, because LESS is based on JavaScript style code.

Define and describes the mechanisms used in either LESS or SASS: mixins, variables, functions, operators, nested variables

LESS, Leaner Style Sheets, is a language that is compatible with CSS. JavScript will convert Less styles to CSS styles, so they are similar languages. It is easy to learn, and Less offers more capabilities than CSS does. Mixins are incredibly helpful for keeping a website theme the same throughout the entire website. This is helpful for calling upon borders, colors, fonts, and sizes. The stylistic values of the mixin properties can be called upon in the style sheet. For variables, this makes it easier so you can set the values at the top of the document instead of resetting for each CSS ID. The width and height can be attached to a variable name, so whenever the variable is called upon, it will pull that variable value. Functions are primarily used for math and converting numbers. You can convert numbers to percentages or decimals. This just saves the designer time because they donâ€™t have to do number conversions. Speaking of conversions, operations are used for math formulas, and it even converts units. You can do arithmetic formulas with cm and mm, or px and cm. There are possibilities of impossible conversions, and therefore the conversion will be ignored. One other major part of LESS is nested variables, which allows you to create multiple style rules for one section. For example, you can put multiple style rules under the body ID tag. This simplifies the code, which is unavailable in CSS.
